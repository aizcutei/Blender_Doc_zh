Autodesk FBX¶

名称

FBX格式

位置

文件 ‣ 导入/导出 ‣ FBX (.fbx)

版本

3.9.1

Blender

2.79

类别

导入 - 导出

作者

Campbell Barton, Bastien Montagne, Jens Restemeier

用法¶

此格式主要用于在不同的应用程序之间交换角色动画，Cinema4D，Maya，Autodesk 3ds Max，Wings3D等应用程序以及Unity3D，虚幻引擎3 / UDK和虚幻引擎4等引擎都支持此格式。

导出器可以将网格修改器和动画烘焙到FBX中，因此最终结果看起来与Blender中的结果相同。

Note

  • 骨骼需要对其方向进行校正（FBX骨骼是-X对齐，而Blender是Y对齐的），这不会影响蒙皮或动画，但在其他应用程序中导入的骨骼会出错。

  • 动画（FBX AnimStacks动画堆栈，Blender动作）**没有链接到** 那个物体，因为没有真正的方法可以知道哪个堆栈用作给定物体，网格或骨骼的 '活动（激活）' 动作。 这功能可能会在将来增强，以便更加智能，但目前并不急于添加这个功能，所以现在目前必须手动将动作关联到物体。

  • 骨架的实例（例如通过复制或组实例）是 不被支持 的。

Note

  • 导入骨骼的方向很复杂，你可能需要使用相关设置，直到获得预期结果。

  • 动画的支持目前是最小的，我们读取所有曲线，好像它们是“烘焙”的曲线（即一组具有线性插值的闭合关键帧）。

  • 导入的动作被关联至其相关物体，骨骼或形变键是初步的胜利。如果你导出一组单个物体，则必须自己重新分配它们。

Note

只保存动画

FBX文件格式支持仅包含take的文件。由你来跟踪哪些动画属于哪个模型。将会导出的动画是Action编辑器中当前选定的操作。要减小文件大小，请关闭导出任何不需要的部分并禁用 所有动作 。对于骨架动画，通常只需启用骨架，这对于该类型的动画是必需的。减少输出量可以使导出和未来的导入更快。通常情况下，每个动作都有自己的名称，但可以强制将当前或唯一的操作命名为“Default Take”。通常确保此选项保持关闭状态。

路径模式

When referencing paths in exported files you may want some control as to the method used since absolute paths may only be correct on your own system. Relative paths, on the other hand, are more portable but mean that you have to keep your files grouped when moving about on your local file system. In some cases, the path doesn't matter since the target application will search a set of predefined paths anyway so you have the option to strip the path too.

自动

在导出位置的子目录中使用文件的相对路径，对于该目录之外的任何目录都是绝对路径的。

绝对

使用完整路径。

相对

在每种情况下使用相对路径（除了在Windows上的不同驱动器上）。

匹配

根据Blender中使用的路径使用相对/绝对路径。

片段路径

只写入文件名并省略路径组件。

复制

在导出时复制文件并使用相对路径引用它。

分批模式

启用后，将每个组或场景导出到文件。

组/场景

选择是否批量导出组或场景导出到文件。请注意，启用“组/场景”时，你无法使用动画选项 当前动作，因为它使用场景数据，并且组未附加到任何场景。另请注意，启用组/场景时，必须在组中包含骨架物体才能使动画动作起作用。

自身目录

启用后，每个文件都会导出到自己的目录中，这在使用 Copy Images 选项时很有用。所以每个目录包含一个模型，包含它使用的所有图像。注意，如果你没有完整的Python安装，则需要完整的Python安装，此按钮将不会显示。

兼容性¶

导入¶

请注意，导入器是新增加的，并且缺少导出器支持的许多功能。

  • 仅限二进制FBX文件。

  • 版本 7.1 或更高。

丢失项¶

  • 网格：形态键。

导出¶

NURBS曲面，文本3D和变形球（融球）在导出时转换为网格。

丢失项¶

缺少以下某些功能，因为FBX格式不支持这些功能，其他功能可能会在以后添加。

  • 物体实例化 - 导出的物体不共享数据，实例化物体将各自用自身数据写入。

  • 材质纹理 - 仅支持texface图像。

  • 顶点形态键 - FBX支持它们，但此导出器尚未编写它们。

  • 动画流体模拟 -- FBX不支持这种动画，但是可以使用OBJ导出器导出文件序列。

  • 约束 - 使用约束的结果将导出为关键帧动画，但约束本身不会保存在FBX中。

  • Dupli-objects（复制物体） - 目前dupli-objects仅在静态场景中写入（禁用动画时）。
