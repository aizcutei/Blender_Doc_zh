动作编辑器¶

The Action Editor is where you can define and control actions. It enables you to view and edit the F-curve data-blocks you defined as Actions in the F-Curve editor. So it takes place somewhere in between the low-level F-Curves, and the high-level NLA editor.

It gives you a slightly simplified view of the F-curve data-blocks (somewhat similar to F-curve shown without handles). The editor can list all Action data-blocks of an object at once.

Each Action data-block forms a top-level channel (see below). Note that an object can have several Constraint (one per animated constraint) and Pose (for armatures, one per animated bone) F-curve data-blocks, and hence an action can have several of these channels.

动作编辑器。

标题栏¶

上一个/下一个层（向下/向上箭头图标）

在NLA 堆栈中堆叠/暂存不同的动作之间切换，而不必去NLA编辑器并离开调整模式 重新进入另一个长条。

点击向上/向下箭头按钮进入NLA轨道上方/NLA长条下方中的动作，该动作在动作编辑器中将被调整。

如果同一图层/轨道上有多个动作/长条带，则只使用最接近当前帧的动作/条带。

操作者将考虑帐户设置以单独查看/编辑动作（即Solo和NLA Muting ）。这样做是为了更容易预览不同的隐藏（暂存）动作。

  • 如果从单独（solo）的NLA轨道到活动动作，则NLA轨道将被静默（不可用），以便可以单独编辑动作。
  • 同样，如果NLA堆栈在编辑动作时被静默，下面的NLA轨道将在启用solo的情况下编辑。
  • 如果它的轨道之间切换，为以前的轨道的solo地位将转移到新的轨道。

Note

当你不编辑NLA Strip使用的动作时，这些仍然有效。如果你只是通常为新动作制作动画，则可以使用“向下箭头”临时跳转到上一个动作，而不会丢失正在处理的新动作，然后使用“向上箭头”回到它，一旦你完成检查其他动作时。

动作

A data-block menu.

添加 +
当一个动作被创建时，它被存储在NLA动作存储（暂存）器中。
取消链接
当 Shift-LMB 点击它清除假用户并且也从NLA堆栈中移除暂存的动作。
按下（双向下箭头图标）
将活动动作添加到NLA堆栈作为作用长条带。这与在NLA编辑器中按下Push Down 按钮基本相同。
藏匿（雪花图标）
将活动动作暂存在NLA堆栈上。即它以与创建新动作相同的方式添加为非贡献（起作用）堆栈。

Note

在这两种情况下（按下和隐藏），一旦该动作被添加到NLA堆栈中，它就被清除/取消分配给活动动作槽（即不能从动作/图形编辑器中进行编辑，除非您输入稍后在相应条上的“调整模式”）。
