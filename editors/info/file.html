文件¶

管理文件的选项有：

新建 Ctrl-N

清除当前场景，加载启动文件startup.blend。

打开 Ctrl-O

打开 blend文件。

打开最近文件 Shift-Ctrl-O

显示 recently （最近）被打开的 .blend 文件列表。

恢复上次会话

这将打开Blender在退出的时候自动保存的 quit.blend 文件。该选项用于 恢复 最后一次偶然关闭的文件。

恢复自动保存

这将打开Blender自动保存的文件来 恢复 场景。

保存 Ctrl-S

Save 当前 blend文件.

另存为 Shift-Ctrl-S

打开文件浏览器来指定文件名和 保存 位置。

保存副本 Shift-Alt-S

Saves 当前文件的副本。

用户设置 Ctrl-Alt-U

在新窗口打开 用户设置 。

保存用户设置 Ctrl-U

保存当前场景和用户偏好到 startup.blend 文件中。

恢复初始设置

恢复默认场景到 初始设置 。

关联 Ctrl-Alt-O

将数据从外部blend文件（库）链接到当前场景。该数据的版本只能在外部库中使用。 关联 和 追加 仅用于加载其他文件中的选定部分。请参阅 Linked Libraries 。

追加 Shift-F1

将数据从外部混合文件追加到当前场景。新数据从外部文件复制，并从中完全取消关联。

导入

Blender可以使用其他图形软件创建的保存为多种文件格式的信息。请参阅 Import/Export。

导出

一般情况下你的工作都保存在了 .blend 文件中，但是也可以把部分工作导出为其他文件格式，这样可以在其他的图形软件中去处理一些其他工作。参阅 Import/Export.

外部数据

外部数据（如纹理图像和其他资源）可以存储在(打包的) .blend文件中，也可以作为单独的文件（解压缩）存储。 Blender通过相对或绝对路径跟踪所有未打包的资源。请参阅 外部数据打包与解包。

自动打包到 .blend 文件

该选项激活文件打包。如果启用，每次保存blend文件时，所有外部文件都将被保存（打包）在其中。

全部打包到 .blend 文件

将所有的外部文件打包到 .blend 文件中。

解压缩到文件中

将所有打包到此 .blend 文件中的文件解包为外部文件。

全部使用相对路径

与当前工程关联的外置文件设置 relative 当前路径。

全部使用绝对路径

将全部外置文件设置为绝对路径。绝对路径具有系统根目录的完整路径。

报告丢失的文件

此选项可用于检查是否存在指向不存在的解包文件的关联。选择此选项后，信息编辑器标题中将显示一条警告消息。如果没有显示警告，则不会丢失外部文件。

查找丢失的文件

如果我们在blend文件中断开关联，此选项将帮助我们解决问题。文件浏览器将显示。选择所需的目录（或该目录内的文件），并在其中执行搜索，递归地在所有包含的目录中进行搜索。在搜索中找到的每个丢失的文件都将被恢复。这些恢复将作为绝对路径完成，所以如果你想要有相对路径，你需要选择 全部使用相对路径 。

Note

恢复的文件可能需要重新加载。你可以一个接一个地完成，或者你可以保存blend文件并重新加载它，这样所有的外部文件都可以一次重新加载。

退出 Ctrl-Q

关闭 Blender 并将文件保存到``quit.blend``中。
